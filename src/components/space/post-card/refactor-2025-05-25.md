# PostCard Component Refactoring

This document tracks the refactoring of the large `PostCard.tsx` component (1052 lines) into smaller, more maintainable components.

## Motivation

The current `PostCard.tsx` component has grown to over 1000 lines, making it difficult to:
- Maintain and debug
- Extend with new features
- Test individual parts
- Understand the component's responsibilities

## Refactoring Strategy

We'll break down the component into smaller pieces using this approach:

1. Extract utility functions to separate files
2. Create custom hooks for stateful logic
3. Extract sub-components for UI sections
4. Update the main component to use these new pieces
5. Optimize performance with memoization

## Detailed Changes

*This section will be updated as each step is completed*

### Step 1: Extract Utility Functions ✅
- Created `src/utils/mediaUtils.ts` containing the following utility functions:
  - `getFileIcon`: Returns icon information based on file type (modified to return icon name and className instead of JSX)
  - `formatFileSize`: Formats byte sizes to human-readable formats
  - `isImageUrl`: Checks if a URL is an image based on extension or MIME type
  - `extractVideoInfo`: Extracts platform, video ID, and thumbnail URL from video links
  - `extractFirstGifAndRest`: Extracts the first GIF from content and returns it separately
- Added proper type definitions and JSDoc comments

### Step 2: Create Types File ✅
- Created `src/features/posts/types/postCard.ts` with the following types:
  - `Author`: Information about a post's author
  - `Category`: Post category information
  - `PostCardProps`: Main props for the PostCard component
  - `CommentAuthor`: Author information for comments
  - `FetchedComment`: Structure of comments from the API
  - `VideoDisplay`: Information for video display
- Updated `src/features/posts/types/index.ts` to export the new types

### Step 3: Create usePostLikes.ts hook ✅
- Created a custom hook to manage post likes state and functionality
- Features:
  - Manages `hasLikedPost`, `optimisticLikeCount`, and `isLikingInProgress` state
  - Provides `handleLikeToggle` function for like/unlike actions
  - Handles optimistic updates for UI responsiveness
  - Integrates with Supabase to persist like status
  - Handles error cases and reverts optimistic updates if needed
  - Provides proper TypeScript interfaces for props and return values

### Step 4: Create usePostComments.ts hook ✅
- Created a custom hook to manage post comments state and functionality
- Features:
  - Manages comments loading, displaying, and creation
  - Handles comment replies and reply targeting
  - Manages optimistic comment counts
  - Provides functions for fetching comments and replies
  - Handles comment likes through a callback
  - Includes proper error handling
  - Provides TypeScript interfaces for full type safety

### Step 5: Create usePostPin.ts hook ✅
- Created a custom hook to manage post pinning (for admin users)
- Features:
  - Manages `optimisticPinned` and `isPinning` state
  - Provides `handlePinToggle` function
  - Integrates with Supabase RPC function for pin/unpin actions
  - Handles optimistic updates and error cases
  - Provides proper TypeScript interfaces

### Step 6: Create usePostMedia.ts hook ✅
- Created a custom hook to manage post media (videos) interaction
- Features:
  - Manages video modal state 
  - Handles selecting and displaying videos
  - Provides functions for handling video click events
  - Includes TypeScript interfaces for type safety

### Step 7: Create PostCardHeader.tsx component ✅

- Features:

### Step 13: Update main PostCard.tsx to use new components ✅
- Completely refactored the main `PostCard.tsx` component
- Key improvements:
  - Reduced the file size from over 1000 lines to about 350 lines
  - Eliminated utility functions by using extracted ones
  - Used all extracted hooks for managing state

### Step 14: Optimize performance with memoization ✅
- Added React.memo to all components to prevent unnecessary re-renders
- Created smaller, memoized sub-components:
  - FileIcon: For rendering appropriate file type icons
  - PollOption: For rendering individual poll options
  - ImageAttachment: For rendering image attachments
  - FileAttachment: For rendering file attachments
  - LinkAttachment: For rendering link attachments
- Memoized complex UI sections with useMemo:
  - renderedPollOptions: For poll data section
  - contentGifDisplay: For GIF content display
  - attachmentsDisplay: For file attachments section
- Added explicit export default for the memoized component
- These optimizations significantly reduce re-rendering of complex UI sections when only one part of the post data changes
