# Architecture Decision Records (ADRs)

This directory contains the project's Architecture Decision Records (ADRs).

## What is an ADR?

An Architecture Decision Record (ADR) is a document that captures an important architectural decision, including its context, consequences, and the decision itself.

## ADR Index

* [ADR-001: Feature-First Organization](./001-feature-first-organization.md) - Organization structure for the codebase
* [ADR-002: Zustand for State Management](./002-zustand-for-state-management.md) - Choice of state management library
* [ADR-003: Space Feed as Root Path](./003-space-feed-as-root-path.md) - Making Space feed the default root path

## Process

When making a significant architectural decision in the project:

1. Copy `template.md` to create a new ADR file
2. Fill in the details of the ADR
3. Submit it for review
4. Once approved, update this index
